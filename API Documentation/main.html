HTTP Requests:

  • PUT: To update a resource
  • POST: To create a resource
  • GET: To get a resource
  • DELETE: To delete a resource

Usual Server Responses:

  • 200 OK - request was successful
  • 201 Created - request successful and resource created
  • 401 Unarthorized - authentication failed or user doesn't have permissions for requested operation
  • 403 Forbidden - access denied

User Auth

POST: For user registration
This endpoint allows users to sign up by creating a new user account. The request should include a valid user object, and validation is performed using the @Valid annotation. Upon successful registration, it returns a success message along with the created user's information.
